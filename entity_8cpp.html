◆ HUD_GetUserEntity()

cl_entity_t CL_DLLEXPORT * HUD_GetUserEntity ( int  index)

This function retrieves a pointer to a cl_entity_t object that describes a specific user entity.

The entity is identified by the specified index. If the index corresponds to a client-side beam entity, it will return a pointer to that beam. If not, it returns NULL.

If you specify negative numbers for beam start and end point entities, then the engine will call back into this function requesting a pointer to a cl_entity_t object that describes the entity to attach the beam onto.

Parameters
indexThe entity index. This value must be greater than 0 and less than or equal to 1. Negative values or indices greater than the valid range will return NULL.
Returns
A pointer to the cl_entity_t object corresponding to the entity at the given index, or NULL if the index is invalid or no beam is associated.

void CL_DLLEXPORT HUD_TxferLocalOverrides ( struct entity_state_s *  state,
const struct clientdata_s *  client 
)

Copies precision origin and other overrides from clientdata to entity_state.

This is used to transfer fields not part of the usual playerstate update but needed for prediction or spectator logic.

The server sends us our origin with extra precision as part of the clientdata structure, not during the normal playerstate update in entity_state_t. In order for these overrides to eventually get to the appropriate playerstate structure, we need to copy them into the state structure at this point.

Parameters
statePointer to the destination entity_state structure.
clientPointer to the source clientdata structure.

Transfers client-side predicted data to the server-acknowledged state.

Used during prediction correction to carry over persistent fields like weapon states or user variables.

Because we can predict an arbitrary number of frames before the server responds with an update, we need to be able to copy client side prediction data in from the state that the server ack'd receiving, which can be anywhere along the predicted frame path ( i.e., we could predict 20 frames into the future and the server ack's up through 10 of those frames, so we need to copy persistent client-side only state from the 10th predicted frame to the slot the server update is occupying.

Parameters
psPredicted entity state (destination).
ppsPrevious predicted entity state (source).
pcdClient-side clientdata (destination).
ppcdPrevious clientdata (source).
wdWeapon data array (destination).
pwdPrevious weapon data array (source).
